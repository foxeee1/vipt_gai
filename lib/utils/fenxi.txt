## 一、注入位置：顶会主流方案 vs 你的实现
### 顶会核心共识：分层分职责稀疏注入，完全解耦无竞争
近两年顶会工作彻底抛弃了「全层注入/重叠注入」的设计，核心逻辑是**匹配ViT的特征学习规律，给不同Prompt分配专属的、无重叠的注入层，严格分工，避免分支竞争**：
| 顶会方案 | 注入位置设计 | 核心逻辑 |
|----------|--------------|----------|
| CVPR2024 Un-Track | 模态专属Prompt→ViT前3层（低层）；共享对齐Prompt→4-6层（中层）；时序运动Prompt→7-9层（高层），完全无重叠 | 低层负责模态纹理细节对齐，中层负责跨模态语义融合，高层负责全局运动建模，层间完全解耦，无梯度竞争 |
| ICCV2025 M3PT | 仅在ViT每个阶段的首尾层注入Prompt，中间层完全保留预训练权重；模态Prompt注入前半段，融合Prompt注入后半段 | 最小化对预训练主干的破坏，仅在特征语义变化的关键节点注入Prompt，避免全层注入导致的特征分布偏移 |
| ICML2025 MPT | 时序运动Prompt仅注入Transformer后3层，完全不碰低层 | ViT高层特征更关注全局语义和运动模式，低层聚焦纹理细节，时序信息无需在低层注入，避免干扰模态特征学习 |

### 你的实现的优势与核心缺陷
✅ 正确的改进：你已经实现了「Consistency独占5-6层、Temporal独占8-9层、仅第7层过渡重叠」的分层解耦，方向100%契合顶会主流逻辑，解决了全层注入的核心问题。
❌ 致命缺陷：
1.  **缺失低层模态专属注入**：没有给RGB/TIR模态专属Prompt分配前3层的固定注入位置，导致低层跨模态基础对齐没做好，中高层的一致性、时序Prompt没有底层特征支撑，泛化性极差；
2.  **重叠层无梯度隔离**：第7层的双分支重叠注入，没有做梯度隔离，导致两个分支在该层的梯度互相干扰、特征污染，出现「1+1<1」的内耗；
3.  **无输入层基础对齐**：没有在Patch Embedding后（输入层）做基础的模态类型Prompt注入，预训练主干无法区分不同模态的Token，后续注入的Prompt效果大打折扣。

### 你可直接参考的优化
1.  补全**分层注入分工**：
    - 1-3层：仅注入RGB/TIR模态专属Prompt，负责低层跨模态纹理对齐；
    - 5-6层：仅注入Consistency Prompt，负责中层跨模态语义融合；
    - 7层：仅做过渡融合，不做双分支同时注入，避免竞争；
    - 8-9层：仅注入Temporal Prompt，负责高层运动建模；
2.  新增**输入层类型Prompt**：给RGB/TIR模态分别学习1个可学习的类型Token，拼接到Patch Embedding序列中，让预训练主干天生区分模态特征，和ViPT原生设计兼容，几乎不增加参数量。

## 二、注入强度：顶会主流方案 vs 你的实现
### 顶会核心共识：弱注入、动态缩放、与模态质量/层深度绑定
ViPT原生的核心优势是**低参微调、弱注入不破坏预训练特征分布**，近两年顶会工作严格遵循这个逻辑，彻底抛弃了强注入设计：
1.  **强度范围严格压缩**：CVPR2024 Un-Track、TPAMI2025 CMDTrack等主流方案，注入强度α的范围严格控制在**0.05~0.2**，和ViPT原生α=0.1完全对齐，公式统一为`x = x_origin + α * prompt_feat`，保证原始预训练特征占绝对主导，Prompt仅做微调修正；
2.  **强度与层深度绑定**：低层注入强度≤0.1（避免破坏预训练的底层纹理特征），高层注入强度上限0.2（高层语义特征容错率更高），符合ViT的特征学习规律；
3.  **强度与模态质量动态绑定**：ICCV2025 FlexTrack提出，通过特征熵值实时计算模态质量，清晰模态（如白天RGB）的注入强度上限0.2，劣质模态（如夜间噪声RGB）的注入强度上限0.05，避免劣质模态的Prompt污染全局特征。

### 你的实现的核心缺陷
你之前的代码中，注入强度`effective_gate = 0.2 + 0.8 * gate`，范围是**0.2~1.0**，是ViPT原生强度的2-10倍，直接导致两个致命问题：
1.  完全破坏了预训练ViT的特征分布，冻结主干失去意义，模型被迫用少量跟踪数据重新学习特征，必然严重过拟合（对应你Tensorboard中15epoch后Loss反弹、IoU暴跌）；
2.  门控的微小差异被放大10倍，弱分支的梯度几乎完全消失，出现「赢者通吃」的单分支主导，三提示完全退化为单提示，甚至更差。

### 你可直接参考的优化
1.  直接修正注入强度范围，和顶会对齐：
    ```python
    # 替换你原有的强注入代码
    effective_gate = 0.05 + 0.15 * gate[:, idx:idx+1].unsqueeze(-1)
    ```
2.  分层设置强度上限：1-3层低层上限0.1，5-9层中高层上限0.2，和层深度绑定；
3.  新增模态质量动态缩放：通过RGB/TIR特征的熵值计算质量系数，熵值越高（噪声越大），该模态Prompt的注入强度越低，实现自适应弱注入。

## 三、多Prompt协同策略：顶会主流方案 vs 你的实现
### 顶会核心共识：分阶段训练+职责硬约束+特征正交化，彻底解决分支内耗
这是你三提示精度上不去的**核心根源**，近两年顶会的多Prompt协同，完全抛弃了「端到端联合训练、无约束自由竞争」的设计，核心是**强制每个Prompt分支做专属的事，学互补的特征，从根源上避免内耗**：
1.  **分阶段训练，避免初始梯度拉扯**
    - ACM MM2025 LRPD提出两阶段训练：第一阶段仅训练模态专属Prompt，让RGB/TIR分支先学到稳定的判别性特征；第二阶段冻结模态分支，训练一致性Prompt做跨模态对齐；第三阶段用1e-4的极低学习率联合微调，彻底避免了初始阶段多分支的梯度反向拉扯；
    - ICML2025 MPT先单独预训练时序Prompt模块，让其先学到稳定的运动建模能力，再接入主干和模态Prompt联合微调，避免时序分支和模态分支的优化目标冲突。
2.  **职责硬约束，每个分支有专属优化目标**
    ICCV2025 M3PT、AAAI2024 BAT等方案，给每个Prompt分支设计了专属的损失函数，而不是仅用最终的定位损失，强制分支职责清晰：
    - 模态专属Prompt：加模态内分类损失，仅负责学习自身模态的独特判别性特征；
    - 一致性Prompt：加跨模态KL散度损失，仅负责对齐双模态的语义分布；
    - 时序Prompt：加帧间特征平滑损失，仅负责建模目标的运动变化规律。
3.  **特征正交正则，强制互补无冗余**
    顶会方案普遍给多Prompt分支加**正交正则约束**，强制模态专属特征和共享特征、时序特征和表观特征互相正交，彻底解决特征冗余、互相污染的问题，保证1+1+1>3的协同增益。

### 你的实现的核心缺陷
1.  **端到端联合训练，无分阶段**：三个分支同时优化，初始阶段梯度互相拉扯，模型根本无法收敛到最优解，甚至出现反向优化；
2.  **无专属损失约束，职责完全模糊**：三个分支的优化目标只有最终的定位损失，导致所有分支都去学习容易拟合的表观特征，而不是互补的模态/时序/一致性特征，出现严重的特征冗余；
3.  **无正交正则，分支内耗严重**：没有强制分支特征差异化，三个分支学到的特征高度重叠，甚至冲突，最终出现「三提示不如双提示、双提示不如单提示」的结果。

### 你可直接参考的优化
1.  **三阶段训练流程，直接套用**：
    - 阶段1（10epoch）：冻结Consistency、Temporal分支，仅训练RGB/TIR模态专属Prompt，让模态特征先收敛；
    - 阶段2（5epoch）：冻结模态专属分支，仅训练Consistency、Temporal分支，让对齐和运动建模能力收敛；
    - 阶段3（15epoch）：解冻所有分支，用3e-4的极低学习率联合微调，配合早停机制，实现三分支协同收敛。
2.  **新增分支专属损失，绑定优化目标**：
    ```python
    # 总损失 = 定位损失 + 分支专属正则损失
    total_loss = giou_loss * 2.0 + l1_loss * 3.0
    # 1. 一致性分支：跨模态KL散度损失
    if 'token_consistency' in inject_intermediates:
        consistency_loss = F.kl_div(rgb_feat.softmax(-1), tir_feat.softmax(-1))
        total_loss += 1e-4 * consistency_loss
    # 2. 时序分支：帧间平滑损失
    if 'temporal_weight' in inject_intermediates:
        temporal_loss = torch.norm(temporal_weight[1:] - temporal_weight[:-1], p=1)
        total_loss += 1e-5 * temporal_loss
    # 3. 正交正则：强制三个分支特征正交，解决冗余内耗
    feat_rgb, feat_tir, feat_consist = branch_feats
    ortho_loss = torch.norm(torch.mm(feat_rgb.T, feat_tir), p=2) + torch.norm(torch.mm(feat_rgb.T, feat_consist), p=2)
    total_loss += 5e-5 * ortho_loss
    ```

## 四、门控与权重分配：顶会主流方案 vs 你的实现
### 顶会核心共识：sigmoid独立门控替代softmax互斥门控，实现真正的协同
近两年顶会工作**100%抛弃了softmax门控**，核心原因是softmax强制多分支权重和为1，天然造成零和博弈、分支互斥，完全违背多Prompt协同的初衷，主流设计如下：
1.  **sigmoid独立门控**：ICCV2025 CVPT、CVPR2025 DMP均采用sigmoid独立门控，每个Prompt分支的权重独立计算，取值范围0~1，互不影响，允许多个分支同时拥有高权重，真正实现协同贡献，在VTAB-1K基准上比softmax门控精度高4%；
2.  **输入感知动态门控**：门控权重不是仅和层深度绑定，而是和输入的模态质量、运动幅度实时关联，比如运动幅度大时自动提升时序Prompt的门控权重，低光场景自动提升TIR模态Prompt的门控权重；
3.  **稀疏化约束**：给门控权重加L1正则，强制模型在不同层仅激活对应的专属Prompt分支，避免所有分支在所有层都激活，减少冗余计算和特征干扰。

### 你的实现的核心缺陷
你之前的代码用了`F.softmax(gate_logits, dim=-1)`，强制三个分支的门控权重和为1，直接造成**零和博弈、分支互斥**：一个分支权重上升，另外两个必然下降，低层Consistency权重0.9，Temporal和模态分支就只剩0.05，完全失去贡献，三提示直接退化为单提示，甚至因为强制竞争导致特征内耗，精度比单提示更差。

### 你可直接参考的优化
1.  **直接替换门控函数，彻底解决互斥问题**：
    ```python
    # 原错误代码
    gate = F.softmax(gate_logits, dim=-1)
    # 替换为顶会主流的独立门控
    gate = torch.sigmoid(gate_logits)
    ```
2.  **修正门控初始化偏置，匹配sigmoid逻辑**：
    把原来的偏置值（如`[2.0, -2.0, -2.0]`）缩小一倍，改为`[1.0, -1.0, -1.0]`，sigmoid(1.0)=0.73，sigmoid(-1.0)=0.27，既保证了分支的初始优先级，又不会完全锁死其他分支，给梯度留了畅通的通道；
3.  **新增输入感知门控**：把模态质量、运动幅度特征拼接到门控的输入中，让门控权重自适应场景变化，而不是仅和层深度绑定。
