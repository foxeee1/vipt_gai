## 一、从曲线定位的5个致命问题（100%对应你的代码缺陷）
### 1. 门控完全锁死，三分支直接退化为单TIR分支，多Prompt架构彻底失效
**曲线铁证**：
- `Consistency/avg_rgb_weight`从0.5直接跌到0，全程躺平，RGB分支完全不工作；
- `avg_tir_weight`从0直接涨到1，全程锁死，TIR分支独占了所有贡献；
- `avg_weight`（一致性分支）直接跌到0，一致性模块完全失效；
- `weight_dist`从stage1的均匀分布，到stage2/3直接挤到0.9，门控完全one-hot化，彻底退化为单分支。

**代码根因**：
1.  门控初始化偏置错误，给TIR分支的初始权重过高，初始阶段就压制了RGB分支；
2.  正则损失反向诱导，模型走了致命捷径：**直接把RGB/一致性分支的门控压到0，只用TIR分支，就能同时满足KL散度、正交正则的约束，还能降低定位损失**，完全违背了多分支协同的初衷。

### 2. 正则损失完全反客为主，模型根本没在学核心跟踪能力
**曲线铁证**：
- `consistency_reg`/`kl_reg`/`ortho_reg`全程主导Loss下降，而核心的`giou`/`l1`/`location` Loss下降极缓、后期剧烈震荡；
- `temporal_reg`/`mask_reg`全程为0，完全没有任何梯度回传，加了等于没加。

**代码根因**：
1.  正则项权重设置过高，模型为了最小化总Loss，优先满足正则约束，而不是优化定位精度；
2.  KL散度、正交正则的计算逻辑完全错误，给了模型走捷径的空间——压死弱分支就能降低正则损失，而不是学习互补特征。

### 3. 时序分支完全失效，没有任何贡献
**曲线铁证**：`Loss/temporal_reg`全程为0，没有任何梯度回传，和你上一版的问题完全一致。

**代码根因**：
时序特征传递的逻辑还是错误的，`prev_features`没有正确传入上一帧的完整特征，同时时序损失没有正确绑定到总Loss里，梯度完全没回传，时序模块全程躺平。

### 4. 梯度爆炸+训练极不稳定，收敛性极差
**曲线铁证**：
- `Consistency/grad_norm`从0持续涨到0.06，翻了6倍，梯度严重爆炸；
- `prompt_mean`剧烈震荡，完全没有收敛趋势；
- 不同run的IoU曲线差异极大，最高0.68、最低0.64，训练完全不可控。

**代码根因**：
1.  梯度裁剪阈值设置过高，门控锁死导致弱分支梯度消失、强分支梯度持续放大，最终爆炸；
2.  没有给不同分支设置差异化学习率，强分支学习率过高，弱分支学习率完全不足以更新。

### 5. 三阶段训练的冻结逻辑完全错误，每个阶段的目标都没达成
**曲线铁证**：
- stage1就出现RGB分支被压死、门控锁死，完全没达成「模态专属特征收敛」的目标；
- stage2一致性分支权重直接跌到0，没达成「跨模态对齐」的目标；
- stage3联合微调完全没提升，性能和stage1持平，早停直接触发。

**代码根因**：
1.  每个阶段的冻结范围错误：stage1不该放开一致性分支的训练，导致初始阶段就出现分支恶性竞争；
2.  stage2冻结了模态分支，但模态分支已经躺平，一致性分支根本没有有效输入，完全学不到东西；
3.  早停逻辑滥用：stage1/2不该早停，必须训满固定epoch确保基础能力收敛，只有stage3需要早停防过拟合。

## 二、分优先级的落地修复方案（按顺序执行，立即可见效）
### 优先级1：10分钟修改，解决门控锁死+正则反向诱导，让三分支都能正常工作
#### 1. 修正门控初始化，从根源避免初始竞争
**修改文件**：`meta_prompt.py` layer_gates初始化部分
```python
# 1-3层（模态专属层）：固定RGB/TIR均等初始权重，一致性分支初始不抢主导
for layer_id in [1,2,3]:
    self.layer_gates[str(layer_id)][-1].bias.data = torch.tensor([0.5, 0.5, -1.0])
    # sigmoid(0.5)=0.62，RGB/TIR初始均等；sigmoid(-1.0)=0.27，一致性分支弱初始化

# 5-6层（一致性专属层）：固定一致性分支高初始权重，模态分支弱贡献
for layer_id in [5,6]:
    self.layer_gates[str(layer_id)][-1].bias.data = torch.tensor([-0.5, -0.5, 1.0])

# 8-9层（时序专属层）：固定时序分支高初始权重，模态分支弱贡献
for layer_id in [8,9]:
    self.layer_gates[str(layer_id)][-1].bias.data = torch.tensor([-0.5, -0.5, 1.0])

# 第7层过渡层：三分支均等初始权重
self.layer_gates["7"][-1].bias.data = torch.tensor([0.0, 0.0, 0.0])
```

#### 2. 把正则权重降2个数量级，让定位损失绝对主导优化
**修改文件**：`vipt.py` 损失计算部分
```python
# 总损失 = 定位损失（绝对主导） + 极弱正则约束
total_loss = giou_loss * 2.0 + l1_loss * 3.0

# 1. 一致性KL散度损失：权重从1e-4降到1e-6，绝对不能超过总Loss的1%
if 'token_consistency' in inject_intermediates:
    consistency_loss = F.kl_div(
        F.log_softmax(rgb_target_feat, dim=-1),
        F.softmax(tir_target_feat, dim=-1),
        reduction='batchmean'
    )
    total_loss += 1e-6 * consistency_loss

# 2. 正交正则损失：权重从5e-5降到5e-6，仅约束Prompt投影矩阵，不约束特征
if 'branch_prompts' in inject_intermediates:
    rgb_p, tir_p, consist_p = inject_intermediates['branch_prompts']
    # 对Prompt的投影矩阵做正交约束，模型无法通过压死特征走捷径
    ortho_loss = torch.norm(torch.mm(rgb_p.T, tir_p), p=2) + torch.norm(torch.mm(rgb_p.T, consist_p), p=2)
    total_loss += 5e-6 * ortho_loss

# 3. 时序平滑损失：权重1e-6，仅做弱约束
if 'temporal_weight' in inject_intermediates:
    temporal_weight = inject_intermediates['temporal_weight']
    if temporal_weight.shape[0] > 1:
        temporal_loss = torch.norm(temporal_weight[1:] - temporal_weight[:-1], p=1)
        total_loss += 1e-6 * temporal_loss
```

### 优先级2：30分钟修改，解决时序分支失效+正则计算逻辑错误
#### 1. 彻底修复时序特征传递逻辑，让时序模块真正生效
**修改文件**：`vit_ce_prompt.py` `_forward_features_meta`方法
```python
def _forward_features_meta(self, x, tir_feat=None, prev_frame_feat=None, is_first_frame=False):
    # 1. 首帧强制清空时序缓存，避免batch维度不匹配
    if is_first_frame:
        self.prev_frame_feat = None
        self.meta_prompt_generator.reset_temporal_cache()
    
    B, N, C = x.shape
    # 2. Patch Embedding后，先做模态类型Token注入（和ViPT原生兼容）
    x = self.patch_embed(x)
    if tir_feat is not None:
        tir_feat = self.patch_embed(tir_feat)
        # 模态类型Token，仅1个可学习参数，几乎不增加参数量
        x = x + self.rgb_type_token
        tir_feat = tir_feat + self.tir_type_token

    # 3. Transformer前向传播，仅在专属层注入对应Prompt
    for i, blk in enumerate(self.blocks):
        # 仅在当前层是注入层时，才调用inject，非注入层完全不碰
        if i in self.inject_layers:
            x_modulated, inject_intermediates = self.meta_prompt_generator.inject(
                current_layer_id=i,
                x_search=x,
                tir_feat=tir_feat,
                prev_features=prev_frame_feat,  # 传入上一帧完整特征，不是当前层中间特征
            )
            x = x_modulated
        # 正常前向传播，绝对不能在循环里覆盖prev_frame_feat
        x, _, _ = blk(x)
    
    # 4. 仅在每帧结束后，保存当前帧的完整特征，给下一帧用
    self.prev_frame_feat = x.detach()
    return x, inject_intermediates
```

#### 2. 修正KL散度计算逻辑，彻底堵死模型走捷径的空间
**修改文件**：`meta_prompt.py` consistency分支计算部分
```python
# 只对目标区域的Token做KL对齐，不是整个特征图，模型无法通过压死一个分支降低损失
def compute_token_consistency(self, rgb_feat, tir_feat, target_mask=None):
    # target_mask是目标区域的二值mask，来自上一帧的跟踪结果
    if target_mask is not None:
        # 仅取目标区域的Token计算KL散度
        rgb_target_feat = rgb_feat[target_mask > 0.5]
        tir_target_feat = tir_feat[target_mask > 0.5]
    else:
        # 无mask时，取前10%的高响应Token（目标区域）
        rgb_norm = torch.norm(rgb_feat, dim=-1)
        tir_norm = torch.norm(tir_feat, dim=-1)
        topk_num = int(rgb_feat.shape[1] * 0.1)
        rgb_topk_idx = torch.topk(rgb_norm, topk_num, dim=1).indices
        tir_topk_idx = torch.topk(tir_norm, topk_num, dim=1).indices
        # 取交集，保证是共同的目标区域
        target_idx = torch.intersect1d(rgb_topk_idx, tir_topk_idx)
        rgb_target_feat = rgb_feat[:, target_idx, :]
        tir_target_feat = tir_feat[:, target_idx, :]
    
    # 计算KL散度，双向对称，避免单向对齐导致一个分支消失
    kl_1 = F.kl_div(F.log_softmax(rgb_target_feat, dim=-1), F.softmax(tir_target_feat, dim=-1), reduction='batchmean')
    kl_2 = F.kl_div(F.log_softmax(tir_target_feat, dim=-1), F.softmax(rgb_target_feat, dim=-1), reduction='batchmean')
    return (kl_1 + kl_2) / 2
```
