# 给代码智能体的【PyTorch深度学习代码安全与异常检查全局强制规则】
## 总要求
你编写/修改的所有PyTorch代码（尤其针对RGBT单目标跟踪、ViPT框架、Prompt生成器、FOMAML元学习场景），必须**100%严格遵守以下所有规则**，从代码底层规避张量形状不匹配、索引越界、设备不一致、NaN/Inf、梯度异常等所有深度学习常见问题。所有规则为强制要求，无特殊说明不得豁免。

---

## 一、前置通用强制规则
1.  所有代码必须适配用户的核心场景：PyTorch框架、RGBT多模态跟踪、ViPT Transformer主干、Prompt学习、FOMAML元学习，所有检查必须贴合该场景的核心痛点。
2.  任何涉及张量操作、索引、维度变换、模型前向、损失计算、梯度更新的代码，**必须先加防护校验，再写核心逻辑**，禁止输出无任何防护的裸代码。
3.  所有校验报错必须包含**精准定位信息**：报错位置、错误原因、预期值、实际值，禁止只抛通用异常，必须让用户一眼定位问题。
4.  所有新增代码必须兼容用户现有ViPT代码结构，不得破坏主干、跟踪头、数据加载器的原生逻辑。

---

## 二、张量形状与维度安全规则（最高优先级，RGBT跟踪第一大坑）
1.  **强制维度断言**：所有张量维度变换、拼接、拆分、注意力计算前后，必须加形状断言，格式为：
    ```python
    # 强制格式，必须包含：预期形状、实际形状、报错位置
    assert x.shape == (B, N, D), f"[LayerName/函数名] 张量形状错误！预期: {(B, N, D)}, 实际: {x.shape}"
    ```
2.  **维度操作安全约束**：
    - 禁止无防护的`view()`/`reshape()`，必须先校验元素总数匹配，再做维度变换；
    - 禁止随意使用`squeeze()`/`unsqueeze()`，严禁删除batch维度，必须显式指定要操作的维度，禁止全局squeeze；
    - 双模态特征（RGB/TIR）拼接/融合前，必须校验两个模态的特征维度完全一致，禁止维度不匹配的广播操作；
    - Prompt注入前后，必须校验主分支`[z_tokens, search_tokens]`的原生形状不被破坏，仅允许特征数值调制，禁止修改token数量、维度顺序。
3.  **batch维度保护**：所有代码必须兼容batch_size≥1的场景，禁止写死batch_size=1的硬编码逻辑，所有操作必须保留batch维度在第0位。
4.  **维度顺序强制统一**：严格遵守PyTorch默认`NCHW`（图像）/`B N D`（token特征）维度顺序，所有OpenCV/PIL读入的图像必须先做维度转换，再输入模型。

## 三、索引与越界防护规则（数据加载、真值框处理重灾区）
1.  **所有索引操作必须先做范围校验**：
    - 图像裁剪、ROI提取、真值框坐标操作前，必须校验坐标的最小值≥0、最大值≤图像宽高，禁止无校验的坐标索引；
    - token序列索引、特征维度索引前，必须校验索引值在张量的维度范围内，禁止硬编码固定索引，必须用变量+范围校验。
2.  **真值框强制防护**：
    - 所有真值框（bbox）在计算、增强、损失计算前，必须用`torch.clamp()`强制限制在图像有效范围内；
    - 禁止出现负坐标、超过图像宽高的坐标，所有数据增强（缩放、裁剪、平移）后的真值框必须重新做范围校验。
3.  **循环/迭代索引安全**：
    - 时序跟踪、元学习内外环循环中，必须校验迭代索引不超过序列/数据集长度；
    - 时序缓存更新时，必须校验历史缓存的形状、长度与当前帧匹配，禁止越界访问历史帧。

---

## 四、数值稳定性与异常防护规则（NaN/Inf、损失/梯度异常重灾区）
1.  **NaN/Inf全流程拦截**：
    - 模型前向的每一个核心模块输出、损失计算结果、梯度更新后，必须加NaN/Inf校验，格式为：
      ```python
      assert not torch.isnan(x).any(), f"[函数名] 张量出现NaN！位置: {__name__}"
      assert not torch.isinf(x).any(), f"[函数名] 张量出现Inf！位置: {__name__}"
      ```
    - 必须在训练代码开头默认开启`torch.autograd.set_detect_anomaly(True)`，自动定位NaN/Inf的产生位置。
2.  **除法/指数操作强制防护**：
    - 所有除法操作，分母必须加极小值epsilon（默认`1e-8`），禁止分母为0的情况；
    - softmax/sigmoid/exp/log操作前，必须对输入做数值范围限制，避免指数爆炸。
3.  **损失与梯度安全约束**：
    - 损失计算后必须校验损失值为正数、在合理范围内（RGBT跟踪总损失默认1~50），禁止损失为负、损失爆炸；
    - 元学习内外环更新、优化器step前，必须加梯度裁剪`torch.nn.utils.clip_grad_norm_()`，默认max_norm=1.0，禁止梯度爆炸；
    - 可训练参数更新后，必须校验梯度不为None、梯度范数在合理范围内，禁止梯度消失。
4.  **门控/权重数值约束**：所有门控权重、注意力权重必须用sigmoid/softmax限制在[0,1]范围内，禁止无界权重导致的特征爆炸。

## 五、设备与数据类型一致性规则（CPU/GPU混用、类型不匹配重灾区）
1.  **设备统一强制规则**：
    - 代码开头必须定义全局统一设备变量`device = torch.device("cuda" if torch.cuda.is_available() else "cpu")`，所有张量、模型参数必须显式指定到该设备；
    - 模型前向、损失计算前，必须校验所有输入张量、模型参数在同一设备，禁止CPU/GPU张量混用；
    - 禁止在训练循环中频繁做CPU/GPU数据迁移，所有数据加载必须提前指定设备。
2.  **数据类型统一强制规则**：
    - 所有模型参数、特征张量、输入图像统一使用`torch.float32`类型，禁止float16/float64/float32混用；
    - 真值框、标签必须使用与特征匹配的数值类型，禁止long类型与float类型直接计算；
    - 所有张量类型转换必须显式指定，禁止隐式类型转换。

## 六、场景专属强制规则（RGBT跟踪+Prompt+元学习专属）
1.  **Prompt模块安全规则**：
    - 所有Prompt生成器的输出，必须校验token数量、特征维度与预期完全一致；
    - Prompt注入Transformer层时，必须校验注入后的特征形状与注入前完全一致，禁止破坏主干特征结构；
    - 多Prompt融合时，必须校验每个Prompt的形状完全匹配，禁止形状不一致的加权融合。
2.  **FOMAML元学习专属规则**：
    - 内环单步更新前后，必须校验生成器参数的数值范围，禁止参数更新爆炸；
    - Support/Query集拆分后，必须校验两个集合的张量形状、数据类型、设备完全一致；
    - 外环元参数更新前，必须校验内环临时参数的梯度有效，禁止无梯度的元更新。
3.  **跟踪推理专属规则**：
    - 每个视频首帧必须重置时序缓存、Prompt生成器状态，禁止跨视频的缓存污染；
    - 逐帧跟踪时，必须校验当前帧特征与历史缓存的形状、设备完全匹配；
    - 预测框输出前，必须做范围校验，禁止输出超出图像范围的无效框。
